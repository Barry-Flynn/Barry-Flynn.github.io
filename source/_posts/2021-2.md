## 三、提交到GitHub
我们需要对两个分支分别提交文件：

### 1、main分支进行博客更新
下面这三句命令将是你以后每次写完文章进行发布到main分支（存放的是你 hexo g 后的生成文件，即静态展示网页）时所需要输入的：
三行命令
```
hexo clean （清理缓存，可选用）
 
hexo g （生成资源文件）
 
hexo d （部署到服务器）
```
其实这时候如果你刷新你的网站看一下，文章已经更新发布了，但我们还最好备份一下hexo分支代码到GitHub上，万一某一天咱们本地的代码出问题了，就可以直接从GitHub上克隆备份。

### 2、hexo分支进行源码备份


下面这三句命令将是你以后每次想备份hexo分支（博客源代码文件）时所需要输入的：
三行命令
```
（添加到缓存区）
git add -A
 
（这次做了什么更改，简单描述下即可）
git commit -m "你的更新描述"
 
（推送至远程仓库）
git push
```
